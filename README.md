<h3>How do I interpret user needs and implement them into a program? How does creating “user stories” help with this?</h3>
</br>
<ul>
  <li>By creating and using User Stories, it provides the opportunity to interpret the user's wants/needs/desires regarding changes and their level of importance. Knowing these stories allows the programmer to ensure they understand the user prior to making the project so there is less of a         possibility of breaking changes going forward.</li>
</ul>

<h3>How do I approach developing programs? What Agile processes do I hope to incorporate into my future development work?</h3>
  <ul>
    <li>First I get as much information as possible from the scrum master, aka my direct manager, and the product owner.</li> 
    <li>After I feel I have a direction to begin, I start small and work my way up.</li> 
    <li>Currently I am working on a project that is somewhat modeled after another project, but not enough to make a direct clone. So I am building it piece by piece from the ground up.</li> 
    <li>Always, test, test, test!</li>
    <li>Get it onto a Staging Server that mimics Production.</li>
    <li>Finally put onto production, and monitor until delivery.</li>
  </ul>
<h3>What does it mean to be a good team member in software development?</h3>
  <ul>
    <li>It means to be open and honest., communicate as well as possible, and always test before committing.</li>
  </ul>
